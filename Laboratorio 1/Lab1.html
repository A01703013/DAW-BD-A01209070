Definición de Términos
Término Significado
1. URL: Por sus siglas significa “Uniform Resource Locator” es una secuencia de caracteres que tiene la finalidad de identificar, localizar y recuperar información determinada en el Internet. Una URL consta principalmente de tres partes: la primera es el protocolo, indicando que tipo de código se va a recuperar (ejemplos de estos son http:// o https://). La segunda parte es el dominio correspondiente al nombre único de dispositovo con el que se está conectando, empieza por el nomre del servidor seguido de las extenciones separadas por puntos. Por último, se encuentra la ruta la cual especifica la lista de carpetas en el dispositivo donde se encuentra el archivo existente [1].
2. MIME: Por sus siglas en inglés, significa “Multipurpose Internet Mail Extensions” que tiene la finalidad de extender la funcionalidad del servicio de correo electronico, de tal mnaera que permite la insercción de documentos como imágenes, sonido y texto.Entre las principales características que añade es el hecho de que permite el envío de múltiples archivos adjuntos en un solo mensaje con longuitud ilimitada, así como el uso de caracteres no pertenecientes al código ASCII. Se basa en el uso de directivas especiales en los encabezados lo cual permite describir el formato utilizado en el mensaje. Los tipos de datos primarios son texto (ejemplo: text/plain [RFC822]), imagen (ejemplo: image/jpeg) audio (ejemplo: audio/basic), video (ejemplo: video/mpeg) y aplicación (ejemplo: application/octet-stream) [2].
3. Método y códigos de error HTTP: Los metodos de http son un conjunto de métodos de petición que sirven para identificar diversas acciones que se desean realizar para un recurso determinado. También son conocidos como HTTP verbs.Por otro lado, los códigos de error también son conocidos como códigos de estado de respuesta, lo cuales indican si se ha completado satisfactoriamente una determinada solicitud HTTP. Estod se clasifican en espuestas informativas, respuestas satisfactorias, redirecciones, errores de los clientes y errores de los servidores [3].
4. XML: Por sus siglas en inglés, significa “Xtensible Markup Language”, el cual es un formato universal para datos y archivos estructurados. De esta forma, XML es un metalenguaje de definición de documentos, el cual está estructurado a través de etiquetas y marcas. Este lenguaje fue desarrollado con el objetivo de lograr páginas web con más semántica, de tal manera que se separara la estructura del contenido propio de la página web. La principal diferencia con respecto a HTML es la capacidad de definir etiquetas que dependen del tipo de dato con el cual se está trabajando [4].
5. XHTML: Es un leguaje muy similar al HTML pero con notables ventajas lo cual hace que sea un leguaje más robusto al momento de diseñar una página web. El princpal objetivo (diferencia) es que busca que se escriba el contenido de tal manera que se exprese qué es cada cosa y no cómo debe de verse cada una de ellas; para ello todas las etiquetas y atributos que definan el aspecto se suprimieron, dejando solo aquellas etiquetas que sirven para definir el significado de los diversos elementos del la pagina [5].
6. HTML 5: Corresponde a la ultima versión de HTML, la cual incluye nuevos elementos atributos y comportamientos. Se caracteriza pro tener un conjunto más amplio de tecnologías que permite a lso sitios web ser más diversos y con mayor alcance. Entre algunas características sobresalientes es el hecho de que permite a las páginas web almacenar datos localmente en el lado del cliente, permitiendo operar sin conexión; de la misma manera, permite una mayor optimización de la velocidad y uso de hardware, entre muchas otras ventajas [6].
7. JS: Es un leguaje de programación que premite la realización de atividad más complejas (funcionalidad) en una página web. De esta manera, permite crear contenido dinñamico, controlar archivos multimedia, crear imágenes animadas, entre más cosas. En otras palabras, permite una funcionalidad de un lenguaje de programación tradicional como almacenar valores, operaciones escritas en formato de texto, entre más funcionalidad [7].
8. PHP: Por sus siglas en inglés, significa “Hypertext Preprocessor”, tiene la característica de ser un lenguaje de programación del lado del servidor independiente de la plataforma, de tal mnaera que se ejecuta en el servido web, justo antes de que se envie la página al cliente. El código PHP se encuentra incustado en el en el script HTML, por lo que el código PHP se encuentra encerrado entre las etiquetas “”, permitiendo entrar y salir del modo PHP [8].
9. JSP: Por sus siglas en inglés, significa “JavaServer Pages”, el cual es una tecnología orientada a crear aplicaciones web que se puedan ejecutar en varios servidores web y en múltiples plataformas. Las páginas web JSP están compuestas de código HTML/XML, en donde se mezclan etiquetas especiales que permiten programar scripts de servidor en sintaxis Java [10].
10. AJAX: Por sus siglas en inglés, significa “Asynchronous JavaScript And XML”, el cual hace refencia a un conjunto de tecnologia para el desarrollo de página web, lo cual hace que sean más responsivas debido a que el intercambio de paquetes entre el servido y las páginas web no se vuelven a cargar cada vez que el usuario realiza un cambio de entrada ( no se vuelve a cargar la página). AJAX sesta compuesto de XHTML y CSS para la presentción de la información, DOM (Documento Object Model) para visualizar e interactuar dinámicamente con la información, XMLHTTPRequest para la manipualción de datos de manera asincónica con el servidor web, XML y XSLT para el intercambio y manipulación de datos y por último JavaScript para enlazar solicitudes e información de datos [11].


Respuesta de las Preguntas

• ¿Cuál es la diferencia entre Internet y la World Wide Web?

Es importante mencionar que Internet y WWW no son lo mismo, es decir, no hacen refecia al mismo significado.Internet es una red de comunicación, mienstras que WWW (web) es un servicio que funciona gracias a internet.De esta manera, el Internet se define propiamente como una red de redes, la cual se logra interconectar diversos dispositivos tecnológicos a lo largo del mundo utilizando protocolos TCP/IP, generando redes heterogéneas que tengan el funcionamien to de una red única, permitiendo de esta manera el intercambio de información en ellos [11].Por otro lado, WWW (World Wide Web) es definido como un sistema de archivos que se encuentran interconectados a los cuales se puede acceder a través de Internet. Dicho de otroa manera, son un conjuto de datos, ya sean textos, archivos, imágenes,etc., agrupados en portales [12].

• ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
  1. GET: sirve para recuperar datos, es decir, solicita la representación de un recurso en específico.
  2. HEAD: es una petición idéntica a la del método GET, sin embargo la diferencia es que esta no devuelve el cuerpo de la respuesta.
  3. POST: es comunmente utilizado para enviar una entidad a un recurso en especifico, generalmente un servidor, lo cual causa a su vez un cambio en el estado del recurso (servidor)
  4. PUT: tiene la finalidad de reemplazar todas las representaciones del recurso de destino con la carga (útil) de la petición generada.
  5. PATCH: es ´únicamente utilizado para generar modificaciones parciales aun recurso.
  6. DELETE: como su nombre lo indica, borra un recurso en específico [13].
• ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Para lograr la interacción entre el usuario y algún sitio web para solicitar la contraseña es posible la utilización de dos métodos, el primero es el método get lo cual estipula que los datos del usuario serán enviados (adjuntados) en la barra de direcciones, al final de la url correspondiente, seguido de un signo de interrogación. En caso de enviar más de un dato, estos se enviarán separados por un “&”. Ejemplo: http://www…………?nombre1=valor1&nombre2=valor2.Por otro lado, el método post no realiza el envío de la información a través de la url, más bien como parte del cuerpo de la petición generada [14].

Se utilizaria el métod GET [13].

• Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

El código 200 es un código de estados de respuesta HTTP, indicando si se ha completado satisfactoriamente o no la solicitud realizadas. La codigos e respuesta de estado se clasifican en 5 clases principales: respuestas informativas, satisfatorias, redirecciones, errores de los servidores y errores de los clientes.En específico, el código 200 (tambien conocido como “OK”), pertenece a grupo de las respuestas satisfacorias, indicando que la solicitud realizada ha tenido éxito, es importante mencioanr que el significado del código varía según el método http empleado.En caso del métdo GET, significa que el recurso se ha obtenido exitosamente y se ha transmitido en el cuerpo del mensaje; HEAD, significa que los encabezados están en el cuerpo del mensaje; PUT o POST, significa que el recurso se transmite en el cuerpo del mensaje [15].

• ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

El código de estado 404 (Not Found) hace refernecia a que el servidor no pudo encontrar el contenido solicitado. Dicho de otra manera, significa que el navegador fue capaz de resolver la petición del número de IP asignado a un dominio y encontrar el servidor requerido correctamente, pero que el servidor no fue capaz de encontrar el fichero solicitado [16]. Dicho lo anterior, es posible mencionar que no es responsabilidad del desarrollador corregir ese error, esto es debido a que es una página web que se ha eliminado, es un enlace externo a una página eliminada o se cambio el nombre de una página. En caso de que la página haya sido eliminada de manera permanente, el error 404 es normal, sin embargo, el contenido de la página eliminada se muestra en en algún otro sitio, es necesario redireccionar desde la URL antigua a la más actual. En caso de que haya sido por un cambio de nombre, es necesario que se cree la redirección 301 desde el archivo “.htaccess”; de este modo se indica tanto a usuarios como motores de búsqueda que el contenido de la página solicitada ha sido transladado a una nueva dirección [17].

• ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

El error 500 (Internal Server Error) hace referncia que el servidor ha encontrado una situación la cual no ha sabido manejarla (no sabe cuál es el problema). Dicho de esta manera, este es un error del lado del servidor, por lo que no es un problema de los dispositivos móviles (no es un prolema del usuario y/o cliente).De esta manera, este error es necesario que sea solucionado por los desarrolladores, ya que comúnmente hacen referencia a un error de permisos, lo cual se debe a un permiso incorrecto en uno o más archivos o carpetas, generalmente en un script PHP o CGI; un tiempo de espera de PHP, en donde al conectarte con recursos externos se genera un agran tiempo de espera; y por útlimo, un error en la codificación en “.htaccess” [18].

• ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Cuando se dice que un elemento está desaporbado (deprecated) significa que el uso de dichos tags o atributos del HTML no es recomendado, ya que ha sido sustituidas por elementos con mayor funcionalidad o más flexibles.Esto sucede generalmente debido a que el estándar HTML es actualizado por lo que diversas características del lenguaje son agregadas, removidas o desaprovada.Ejemplos de ciertos elementos depreciados son: applet, basefont, center, dir, menu; así como una gran cantidad de atributos (align, alink, background, bgcolor, clear, compact, color, border, entre otros) [19].

• ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

En la versión HTML 5 se incorporan nuevas etiquetas (elementos) como: SECTION, ARTICLE, ASIDE, HEADER, FOOTER, NAV, DIALOG, entre otros. De la misma manera, incorpora etiquetas específicas para agregar audio y video de forma directa en la página de internet. De la misma manera, permite la vinculación con API’s cada vez más complejas que permiten crear página más complejas [20]. De la misma manera, una característica específica de HTML 5 es que está orientdo a 2 públicos distintos, desarrolladores y usuarios; de esta manera, mientras que a los desarrolladores se le presentan especificaciones más estrictas, para los usuarios se le presentan especificaciones más amplias, de tal manera que se permita la compatibilidad con verciones anteriores de HTML [21].

• ¿Qué componentes de estructura y estilo tiene una tabla?

Una tabla puede tener celdas de encabezado, para ello se utiliza el elemento "th" que genera una celda encabezado en la primera fila de la tabla.El stributo scope sirve para indicar a qué celdas afecta una celda encabezado.Otro componente que aporta gran legilibilidad es el elemento "caption", el cual genera un título para proveer una descripción concisa de la información que presenta la tabla.Por otro lado, también es posible la iunificación de celdas adyacentes para ello se utilizan los atributos colspan, para concatenación horizontal; y rowspan, para concatenación vertical.De la misma manera, existen ciertos elemetos que permiten una mejor organización de la información, estos son "thead", utilizado para presentar la información del encabezado; "tbody", utilizado para agrupar las filas que representan la información contenida en la tabla; y "tfoot", que contiene las filas que representa un resumen de la tabla.Por último, las columnas de la tabla pueden ser concatenadas con el elemento "colgroup", el cual sirve para agrupar columnas que tengan significados similares [22].

• ¿Cuál es el ciclo de vida de los sistemas de información? ¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de vida y de desarrollo de un sistema de información consta de 6 fases principales.La primera es la investigación preliminar la cual consiste en tener conocimineto de la organización, de sus necesidades y de los problemas a los que se enfrenta. En esta fase se realiza toda la investigación necesaria para la obtención de la información. La fase 2 es el análisis de la información en donde con la información recaudada, se generan gráficos, mapas mentales y diagramas para mostrar una forma condensada de la información. La fase 3 consiste en el diseño del nuevo sistema el cual debe de representar una avance progresivo de la versión anterior, de tal manera que el usuario logre familiarizarse con los nuevos procedimientos; en este punto se da una posible solución a las necesidades del cliente, este aspecto también está relacionado con la apariencia del sistema. La cuarta fase es el desarrollo y la documentación del nuevo sistema, en dicha fase se inicia la programación del nuevo sistema de tal manera que todo aquello que se realice en código esté respaldado por la documentación pertinente. La quinta fase es la implementación del sistema de información, la cual consiste en la fase práctica del sistema, de tal manera que se detectar las posible fallas que pueda tener; el objetivo fundamental de esta fase es la detección de errores. La última fase es el mantenimiento del sistema el cual consiste en procurar un perfecto funcionamiento del sistema mientras este esté en uso; de la misma manera, en esta fase se da solución a los problemas que ocurran con la operatividad y las nuevas exigencias de los usuarios [23].
